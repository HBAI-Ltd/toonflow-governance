# Toonflow コミュニティガバナンス憲章

**v1.0 · 2026年3月24日 · HBAI-Ltd**

---

## 第1章 総則

本憲章は、Toonflowコミュニティ理事会の構成、権利、義務、およびコミュニティファンドの利益分配メカニズムを規定する。

---

## 第2章 コミュニティ理事会

### 2.1 構成

コミュニティ理事会は計 **15議席** で構成される：

| 議席の種類 | 数 | 選出方法 |
|---------|------|---------|
| コミュニティ貢献者枠 | 11議席 | 全有効貢献者による選挙で選出 |
| HBAI任命枠 | 4議席 | HBAI-Ltdの内部任命で選出 |

### 2.2 貢献者枠の立候補・選挙ルール

**立候補資格：**
- 年間貢献ポイントランキングの上位30名以内であること
- 過去の累積ポイントが 200ポイント以上であること（短期的なポイント稼ぎを防止するため）

**選挙ルール：**
- 選挙権：年間ポイントが 50ポイント以上の全貢献者に投票権が付与される
- 1人最大11票まで投票可能とし、得票数上位11名が当選となる
- 各候補者の得票数を含め、選挙結果は一般に公開・公示される

**任期：**
- 任期は1年とし、任期満了後に再選を目指すことができる
- 同一人物の連続就任は **2期** までとする（議席の固定化を防止するため）

**欠席の扱い：**
- 正当な理由なく連続2回理事会を欠席した場合、その年のポイント第12位の者が自動的に繰り上げ当選となる
- 自主的に辞任する場合は30日前までに書面で通知する必要があり、ポイントの次点者が繰り上げ当選となる

### 2.3 HBAI任命枠のルール

- HBAI-Ltd内部で任命され、会社の戦略的方向性を代表する
- 任命リストは毎年公開・公示される
- HBAIの4議席は **最終拒否権（Veto）** を保持する（プロジェクトのコアな方向性とコミュニティの基本利益を保護するため）
- 拒否権を行使する場合は、公開される議事録にその理由を明記しなければならない

## 第3章 理事会の権利と義務

### 3.1 権利

| 権利 | 説明 |
|------|------|
| **コミュニティファンドの優先順位に関する投票権** | 四半期ごとにファンドの使途方向について投票を行い、過半数の賛成で実行される |
| **年間利益分配案の提案権** | 理事会は利益分配案を提案でき、HBAIの最終確認を経て決定される |
| **ロードマップを知る権利** | Toonflowの製品ロードマップを事前に把握し、公開前に議論に参加できる |
| **新機能のクローズドベータテスト優先参加権** | 新バージョンのクローズドベータテストの参加資格を優先的に獲得できる |
| **公式代表としての資格** | 理事会メンバーは「Toonflowコミュニティ理事」として外部イベントに参加できる |

### 3.2 義務

| 義務 | 説明 |
|------|------|
| **参加義務** | 四半期ごとの理事会には、特別な事情がない限り出席しなければならない（オンライン/オフライン問わず） |
| **公示義務** | 各会議後7営業日以内に、会議の要約と投票結果を公開・発表する |
| **中立義務** | 投票時はコミュニティ全体の利益を優先し、個人的または商業的利益を代表して利益供与を行ってはならない |
| **守秘義務** | ロードマップなどの未公開情報は、公式発表前に外部に漏洩してはならない |
| **行動規範** | Toonflowコミュニティ行動規範を遵守する。理事会メンバーの公的な言動はコミュニティのイメージを代表する |

### 3.3 利益分配（配当）について

- **理事会の議席自体には利益分配を受け取る資格は付随しない** —— ガバナンス権と経済権は完全に分離される
- 理事会メンバーが個人の貢献ポイントを保有している場合、**一般の開発者として**コミュニティファンドの利益分配プールに参加し、ポイントは議席の身分によって加重または減算されることはない
- HBAI任命枠のメンバーにも上記のルールが同等に適用される

## 第4章 コミュニティ貢献ポイントシステム

### 4.1 ポイントシステムのステータス

> **ステータス：進行中、さらなる議論待ち**
>
> 貢献ポイントシステムの細則（ポイントの計算方法、どの行動がポイントに加算されるか、利益分配の要件などの詳細）については、理事会での議論の結果、より十分な検討とコミュニティからのフィードバックが必要であると判断された。
>
> **次回の理事会までに**細則の草案を完成させ、その際に改めてコミュニティに公示し、意見を募集する。

### 4.2 ポイントシステムのフレームワーク（細則確定後に発効）

確定後は、以下の次元から設計を行う：

- **ポイント獲得の次元：** コード貢献、エコシステム貢献、コミュニティ貢献
- **利益分配の参加要件：** 理事会での議論により決定
- **ポイント計算ルール：** 理事会での議論により決定
- **ポイントの照会と公示：** ポイントの照会は公開され透明性が保たれており、誰でもリアルタイムで確認できる

> 詳細なポイントルールは、最終的に公示されるバージョンを正とする。理事会は、この期間中、コミュニティが積極的に意見を提出することを奨励しており、GitHub IssueやDMを通じたフィードバックを受け付けている。

## 第5章 コミュニティファンド利益分配メカニズム

### 5.1 資金源

商業ライセンス収入全額の **50%** がコミュニティファンドに充てられ、四半期ごとに配分・公示される。

### 5.2 利益分配資格

- **参加要件：** 年間ポイント ≥ 50ポイント
- **分配方法：** 各個人の年間ポイントが全有効貢献者のポイント総和に占める割合に応じて分配する
- **分配対象：** 要件を満たす全貢献者。理事会メンバーも含む（個人のポイントで参加し、議席による加重はなし）

### 5.3 利益分配の計算例

> ある年を想定した場合：
> - コミュニティファンド利益分配プール：20万円（または相当額）
> - 要件（≥ 50ポイント）を満たす貢献者：40名
> - 全有効貢献者の年間ポイント総和：10,000ポイント
> - ある貢献者の年間ポイント：500ポイント
>
> **この貢献者の配当額 = 20万 × (500 / 10,000) = 1万円**

### 5.4 透明性メカニズム

| 頻度 | 内容 |
|------|------|
| 四半期ごと | ライセンス収入総額 ＋ コミュニティファンド配分額の公表 |
| 半年ごと | ファンド使用明細の公示 |
| 毎年 | 利益分配リストの公示（各個人のポイント、配当額を含む。本人は匿名を選択可能） |

## 第6章 移行措置

第1回正式選挙が完了するまでの間（予定として6ヶ月以内）：

- ライセンスに関する意思決定権は、暫定的に創設理事会（創設時の任命制）が代行する
- 法人主体および法的責任は、暫定的に HBAI-Ltd が担う
- ライセンス契約は、HBAI-Ltd を契約主体として締結する
- コミュニティオブザーバー（立会人）は、ライセンスに関するすべての意思決定の議論に同席する
- 選挙完了後、ライセンス権限は正式に理事会へ移譲され、法人主体の変更条件については理事会の投票により決定される

## 第7章 憲章の改訂

- 本憲章の改訂には、理事会での投票による可決（貢献者枠11議席中、7議席以上の賛成） ＋ HBAI の最終確認が必要である
- 改訂内容は14日前に草案を公示し、コミュニティからのフィードバックを受け付ける必要がある
- 各改訂ごとにバージョン番号と変更履歴（改訂内容）を記録し、過去のバージョンはアーカイブとして公開する
